# derivation_checks —— 推导机械检查流程

> 收到任何推导时,**绝不"凭感觉"判对错**。逐项执行下列检查,把每项结果列出来。
> 输出格式:每步 ✓ / ✗ + 简述;发现问题给出**具体位置和建议**,而非笼统"看起来对"。

---

## 检查清单(逐项执行,逐项报告)

### 1. 量纲检查
- 每个等式两边量纲是否一致。
- 加速器领域高频出错点:
  - 能量单位混用 eV ↔ J(尤其代入 SI 公式时漏乘 e)。
  - **归一化发射度 εₙ = βγ·ε 与几何发射度 ε** 混用;单位 m·rad vs mm·mrad。
  - 角频率 ω 与频率 f(差 2π);波数 k 的定义。
  - 高斯单位制 ↔ SI 制混用(经典电子半径、空间电荷项常踩)。
- 报告:列出可疑等式,指出哪一侧量纲对不上。

### 2. 极限 / 特例检查
- 令关键参数趋 0 或趋 ∞,结果是否退回已知简单情形:
  - 空间电荷项 → 0,应退回单粒子线性运动(回到 Hill 方程 / 裸调谐)。
  - 流强 I → 0,集体效应消失。
  - γ → ∞,空间电荷力 ∝ 1/γ² 应被压制。
  - 聚焦梯度 → 0,束应自由漂移发散。
- 报告:每个极限是否给出物理上正确的退化结果。

### 3. 守恒 / 符号检查
- 该守恒的量是否守恒:能量、动量、相空间体积(**Liouville**)、绝热不变量。
- 符号约定是否自洽:聚焦/散焦符号、漂移方向、坐标系(x-x' 相空间旋转方向)。
- 报告:指出任何隐性破坏守恒律的步骤(常见于人为引入耗散却未说明机制)。

### 4. 符号代数验证(不靠心算)
- 调用 `scripts/check_algebra.py`(sympy),把推导关键步骤真算一遍化简 / 求导 / 积分。
- 对比模型化简结果与用户给的结果是否符号等价。
- 报告:贴出 sympy 的判定(等价 / 不等价 + 差异项)。
- > 脚本尚未实现时(Step 3 前):必须显式声明"代数未经 sympy 验证,以下为人工核对",
  > 不得把心算结果当作已验证。

### 5. 数量级检查
- 代入典型数值,看结果量级是否物理合理。参考典型范围:
  - 储存环束流能量:MeV ~ 数百 GeV;直线对撞机到 TeV。
  - 发射度:第三代光源 nm·rad 量级;衍射极限环 pm·rad。
  - 流强:mA ~ A;单束团电荷 nC 量级。
  - 调谐移 |Δν|:空间电荷致 incoherent tune shift 常在 0.1 量级即危险。
- 报告:给出代入的数值与得到的量级,标注是否落在物理合理区间。

## 输出模板

```
推导核查结果:
1. 量纲     ✓/✗  —— ……
2. 极限特例 ✓/✗  —— ……
3. 守恒符号 ✓/✗  —— ……
4. 代数(sympy) ✓/✗/未验证 —— ……
5. 数量级   ✓/✗  —— ……

结论:<在第 N 步发现问题:具体位置 + 建议>;或 <五项均通过,未发现机械错误(注意:机械检查通过不等于物理建模正确)>。
```

> 提醒:机械检查能抓住量纲/代数/守恒类错误,**不能**保证物理假设本身成立。
> 通过后若仍涉及建模假设,补一句来源(走 reference_locator_policy 的降级链)。
